# Vireon AI Beta Rollout Plan

## Goal
Launch Vireon AI to a controlled group of early users, verify core product quality, payment reliability, generation reliability, and user interest before public launch.

## Phase 1 - Internal testing
Target: 2-3 trusted testers

Checklist:
- [ ] Create admin account
- [ ] Create Creator A account
- [ ] Create Creator B account
- [ ] Test image generation
- [ ] Test video generation
- [ ] Test image-to-video
- [ ] Test multi-scene video project
- [ ] Test export final video
- [ ] Test credit purchase
- [ ] Test refund flows
- [ ] Test notifications
- [ ] Test public gallery
- [ ] Test mobile responsiveness

## Phase 2 - Small private beta
Target: 10-20 users

Focus:
- onboarding clarity
- generation quality
- payment confidence
- mobile usability
- bugs/crashes
- confusing UI areas

## Phase 3 - Wider beta
Target: 50-100 users

Focus:
- cost per generation
- conversion to paid credits
- repeat usage
- gallery activity
- creator sharing
- support requests

## Success metrics
- At least 70% of testers complete one image generation
- At least 30% try video or image-to-video
- At least 10% publish something to gallery
- Payment test succeeds without manual intervention
- No critical worker/payment failures

## Recommended rollout order
1. Internal testing
2. Fix critical bugs
3. Private beta with 10-20 users
4. Review cost/payment/generation data
5. Improve onboarding and pricing
6. Wider beta
7. Public launch
